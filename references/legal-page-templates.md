# Legal and Required Page Handling

## Purpose

Create practical required pages while avoiding false legal certainty.

## Rules

- Prefer user-provided legal text for privacy policy and terms/user agreement.
- If the user does not provide legal text, generate a clearly labeled draft.

## Contact Page

Include:

- Company or brand name
- Email, phone, or contact link
- Business address if provided
- Service hours if provided
- Short contact form only if it is static and clearly points to email or an external form provider

If contact data is missing, use "å¾è¡¥å" placeholders.

## Privacy Policy Page

If user provides content:

- Use their text with light formatting only.
- Do not rewrite legal meaning unless asked.

If content is missing:

- Create sections for information collected, usage, sharing, retention, user choices, minors, updates, and contact.
- Label it as a draft requiring review.
- Use placeholders for company name, contact email, jurisdiction, and effective date.

## Terms or User Agreement Page

If user provides content:

- Use their text with light formatting only.

If content is missing:

- Create sections for service description, acceptable use, intellectual property, disclaimers, limitation of liability, termination, governing terms placeholder, and contact.
- Label it as a draft requiring review.
- Use placeholders for company name, contact email, jurisdiction, and effective date.

## Final Handoff Checklist

Before final handoff, report:

- Privacy policy source: user-provided / generated draft / missing
- Terms source: user-provided / generated draft / missing
- Contact page complete: yes / no
- Remaining placeholders
